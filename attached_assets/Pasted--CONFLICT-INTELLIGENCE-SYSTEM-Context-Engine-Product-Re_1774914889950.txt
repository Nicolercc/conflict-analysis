1. Executive Summary
Conflict Context Engine is a web application that transforms raw news articles about geopolitical conflicts, military strikes, and security incidents into structured, visually-rich intelligence briefs — instantly and automatically.
A journalist, researcher, student, or news consumer pastes a single article. Within seconds they receive: a live world map with animated strike markers, a timeline of 2–3 related historical events plotted on the same map, an AI-generated plain-language summary, a source credibility score with explanation, and an escalation risk assessment.
The product addresses a real gap: conflict news is fragmented, decontextualized, and overwhelming. Context Engine makes it spatial, structured, and accessible — turning one input into a complete intelligence picture.

Hackathon Theme Fit: This product lives at the intersection of AI, journalism, and civic tech. It is demo-friendly (visual, immediate feedback), technically impressive (real-time AI + geospatial rendering), and emotionally resonant (the wars in Ukraine, Gaza, Sudan are in the news cycle daily).


2. Problem Statement
2.1 The Problem
Conflict reporting is broken for the average reader. A single missile strike article contains no map, no historical context, no credibility signal, and no sense of whether the situation is escalating. Readers are left to piece together meaning across dozens of tabs.

For journalists and researchers, the problem is the opposite: too much fragmented data, not enough synthesis. Verifying a report against related events, cross-referencing actors, and locating incidents geospatially requires significant manual effort.

3. Product Vision & Goals
3.1 Vision Statement
"One article in. Full Vantage brief out — in under 10 seconds."

3.2 MVP Success Criteria
	•	User can paste any English-language conflict article and receive a complete brief
	•	World map renders with correct animated strike marker at extracted location
	•	2–3 historically related events appear on map as secondary markers with dashed connection lines
	•	Summary types out character-by-character (typewriter effect) on results render
	•	Credibility score animates from 0 to final value
	•	Escalation bar fills with appropriate color (green/yellow/red) based on risk level
	•	Demo article pre-loaded for hackathon judges
	•	Full flow completes in under 12 seconds on standard connection

3.3 Non-Goals (MVP Scope)
	•	Real-time news monitoring or RSS ingestion
	•	User accounts, saved briefs, or history
	•	Multi-language support
	•	Mobile-native app
	•	Database persistence
	•	Advanced source verification (rule-based only for MVP)


4. Feature Specification
4.1 Article Input
FEATURE ID: F-01
	•	Large textarea accepting free-text paste of any news article
	•	Character count displayed in real-time
	•	"Load Demo Article" button pre-fills a Kyiv missile strike article for demonstration
	•	Analyze button activates only when textarea has content (min 50 chars)
	•	Input field styled with focus glow effect on interaction

4.2 Analysis Pipeline (AI)
FEATURE ID: F-02
	•	On submit, article is sent to Anthropic Claude API (claude-sonnet-4-20250514)
	•	System prompt instructs model to return structured JSON only — no markdown, no preamble
	•	JSON schema includes: headline, location (city/country/lat/lng), summary, actors array, credibility object (score/label/reason), relatedEvents array (3 items with lat/lng/type/date/title/description), escalationRisk, escalationReason
	•	Response parsed and validated client-side; error state shown on parse failure
	•	Full pipeline target: under 8 seconds

4.3 Radar Sweep Loader
FEATURE ID: F-03
	•	Replaces input view during analysis
	•	SVG radar animation: rotating sweep arm with faint green trail
	•	Progress steps cycle through: Parsing article → Extracting location → Correlating events → Scoring source → Generating brief
	•	Step indicator bar fills segment by segment as phases advance (750ms intervals)

4.4 World Map with Animated Markers
FEATURE ID: F-04 — HIGHEST PRIORITY
	•	SVG world map rendered at 960x500 viewBox, Equirectangular projection
	•	Continent shapes drawn as SVG path elements with dark green fill on near-black background
	•	Latitude/longitude grid lines at 30-degree intervals
	•	Main incident marker: double expanding pulse rings in red, solid core dot with white center, SVG filter glow
	•	Markers animate sequentially: main marker appears at 400ms, connection lines at 1000ms, related markers stagger at 1400/1850/2300ms
	•	Related event markers: color-coded by type (strike=red, escalation=orange, negotiation=green, humanitarian=blue, political=purple)
	•	Dashed animated lines connect related event markers to main incident location
	•	Location label overlay (city, country, lat/lng) fades in at bottom-left of map
	•	Type legend appears in top-right corner as related markers are revealed

4.5 Typewriter Summary
FEATURE ID: F-05
	•	AI-generated 2–3 sentence summary renders character by character at 15ms/char speed
	•	Blinking cursor ▌ appears at end of in-progress text
	•	Cursor disappears when typing completes

4.6 Credibility Score
FEATURE ID: F-06
	•	Large animated number counts up from 0 to final score over 1.4 seconds
	•	Color: green (#4ade80) for 75+, yellow (#fbbf24) for 50–74, red (#ff6b6b) below 50
	•	Thin progress bar fills to score width with matching color and glow
	•	Text label (Low / Medium / High) displayed beside score
	•	One-sentence AI explanation of credibility assessment shown in italic

4.7 Escalation Risk Meter
FEATURE ID: F-07
	•	Three levels: Low (25% fill, green), Medium (58% fill, yellow), High (90% fill, red)
	•	Bar fills with CSS transition (1.5s cubic-bezier ease) on component mount
	•	Large risk label (LOW / MEDIUM / HIGH) rendered in matching color with text shadow glow
	•	One-sentence AI escalation reasoning displayed below bar

4.8 Related Event Timeline
FEATURE ID: F-08
	•	Three event cards rendered as a vertical timeline with colored dot-and-line connector
	•	Each card shows: date, event type badge (color-coded), title (serif font), one-sentence description
	•	Cards stagger in with opacity + translateY animation at 200ms intervals
	•	Event types: strike, escalation, negotiation, humanitarian, political — each with distinct color

4.9 Live Ticker
FEATURE ID: F-09
	•	Rotating status messages cycle across top bar every 3.5 seconds with fade transition
	•	LIVE badge in red with pulse animation
	•	Messages: CONFLICT CONTEXT ENGINE · LIVE INTELLIGENCE SYSTEM, GEOSPATIAL MAPPING · EVENT CORRELATION · CREDIBILITY SCORING, PASTE ANY CONFLICT ARTICLE TO BEGIN ANALYSIS, INCIDENT TRACKING · PATTERN RECOGNITION · BRIEF GENERATION

6.4 API Contract — Anthropic
REQUEST
POST https://api.anthropic.com/v1/messages
{
  model: "claude-sonnet-4-20250514",
  max_tokens: 1000,
  system: SYSTEM_PROMPT,
  messages: [{ role: "user", content: `Analyze:\n\n${article}` }]
}

RESPONSE JSON SCHEMA (STRICT)
{
  headline: string,                    // 8-10 words
  location: {
    city: string,
    country: string,
    lat: number,                       // decimal degrees
    lng: number
  },
  summary: string,                     // 2-3 sentences
  actors: string[],                    // 2-4 key parties
  credibility: {
    score: number,                     // 0-100
    label: "Low" | "Medium" | "High",
    reason: string                     // 1 sentence
  },
  relatedEvents: Array<{
    date: string,                      // "Mon YYYY"
    title: string,
    description: string,               // 1 sentence
    type: "strike"|"escalation"|"negotiation"|"humanitarian"|"political",
    lat: number,
    lng: number
  }>,                                  // exactly 3 items
  escalationRisk: "Low" | "Medium" | "High",
  escalationReason: string
}

6.5 Geo Projection Utility
LIB/GEO.JS
// Equirectangular projection: 960x500 SVG viewBox
export function latLngToXY(lat, lng) {
  return {
    x: ((lng + 180) / 360) * 960,
    y: ((90 - lat) / 180) * 500,
  };
}


7. Component Specification
7.1 WorldMap
PROPS
main: { city, country, lat, lng }
related: Array<{ lat, lng, type, date, title, description }>
active: boolean  // triggers animation sequence

ANIMATION SEQUENCE (MS OFFSETS FROM ACTIVE=TRUE)
	•	0ms: Component mounts, all markers hidden
	•	400ms: Main marker fades in with pulse rings
	•	1000ms: Dashed connection lines appear (showLines = true)
	•	1400ms: Related marker 0 revealed
	•	1850ms: Related marker 1 revealed
	•	2300ms: Related marker 2 revealed
	•	2300ms+: Legend appears in top-right

7.2 RadarLoader
No props. Self-contained. Manages its own step state with setInterval at 750ms. Renders SVG radar with CSS spin animation on sweep arm. Step indicator bar fills left to right.

7.3 useTypewriter Hook
function useTypewriter(text: string, speed: number = 15)
returns { out: string, done: boolean }

Resets and restarts whenever text changes. Uses setInterval to append one character per tick. Clears interval and sets done=true on completion.

7.4 useAnimCounter Hook
function useAnimCounter(to: number, dur: number = 1400)
returns number

Uses requestAnimationFrame with timestamp tracking. Linear interpolation from 0 to target over duration. Returns rounded integer on each frame.

7.5 Ticker
No props. Array of 4 message strings cycles on 3500ms interval. Fade transition via opacity CSS. LIVE badge uses CSS blink animation at 2.4s infinite.

7.6 EventCard
{ ev: EventObject, idx: number, active: boolean }

Delay = idx * 200ms. On active=true, setTimeout triggers opacity + translateY transition. Color derived from TYPE_CFG constant keyed by ev.type.

8.3 System Prompt
The system prompt must be sent verbatim to the API on every request. It must specify: return only valid JSON, no markdown fences, no preamble, no explanation, and provide the exact JSON schema with all field types and constraints listed.

9.4 API Wrapper
// lib/api.js
export async function analyzeArticle(article) {
  const res = await fetch('https://api.anthropic.com/v1/messages', {
    method: 'POST',
    headers: { 'Content-Type': 'application/json' },
    body: JSON.stringify({
      model: 'claude-sonnet-4-20250514',
      max_tokens: 1000,
      system: SYSTEM_PROMPT,
      messages: [{ role: 'user', content: `Analyze:\n\n${article}` }],
    }),
  });
  const data = await res.json();
  const raw = data.content?.map(b => b.text || '').join('') || '';
  return JSON.parse(raw.replace(/```json|```/g, '').trim());
}
